User Manual

Overview:

The purpose of the Description Logic Visualizer is to represent description logic in an easy way to quickly grasp the concepts and their interactions with one another in the created world. Due to the constraints of being able to only show so much information, the visualizer is limited by specific constraints on how the knowledge is to be entered and how it can be represented.

Getting Started:

Upon startup, the program’s world state is empty meaning that there are no individuals, concepts, or roles. The concept and role boxes act as the world’s assertion box by allowing the input of known concepts and roles. The T-Box allows the user to input subset and equivalence relations for the visualizer to interpret.

Using Concepts:

The concepts acts as propositions or basic facts that state that a node is an element of some set.

 

Syntax:   Concept(Object)

Example: Person(Chase)

 

Creates a set labeled “Person” with a node “Chase” inside.

 

Using Roles:

The roles act as a way of connecting two nodes or individuals through declared roles, as such “R(a,b)”. Our systems works with a closed world assumption (CWA) so if there is no role explicitly stating that two objects are related it will not be visualized.

 

Syntax: Role(Object1,Object2)

Example: Friends(Chase, Jeremey)

 

Connects the two nodes (Chase, Jeremey) by an arrow labeled “Friends”.

 

Using T-Box:

The terminology box (T-Box) allows for the user to classify concepts to be subsets of other concepts (C⊑D). By specifying a subset relation, the visualizer essentially moves concept C within concept D. (∀x(x ∈ C ⇒ x ∈ D))

 

Subset:

Syntax: ___ SUBSET ___

Example: Male SUBSET Person

 

Equals:

Syntax:   ___ EQUALS ___ AND ___

___ EQUALS ___ OR ___

JSON Formatting:

This folder contains all of the preloaded example files. Each file is a json document which consists of three arrays: concepts, roles, and tbox. These arrays correspond to the fields in the application. Prior to adding a new example, make sure that you use JSONLint (http:/www.jsonlint.com) to check the formatting.

 

Quick Example:

[{

"concepts": [

"Pizza(AllMeat)",

"Topping(Pepperoni)",

"Sauce(Tomato)",

"VeggieTopping(Mushroom)"

],

"roles": [

"HasTopping(AllMeat,Pepperoni)"

],

"tbox": [

"VeggieTopping SUBSET Topping"

]

}]

 

Uploading File:

  1. Click on “Upload” tab.
  2. Choose file to upload. (Ensure file is correctly formatted.)
  3. Click refresh to see changes.

Common Problems:

  • If the visualizer is not displaying anything at all, make sure that the “Refresh” button has been clicked.
  • If the visualizer is not displaying the concepts or roles properly, it may be the case that leading or trailing whitespace within a role is giving the parser problems. An example is “R(a , b)”, leading to the visualizer interpreting the individual as “a ” or “ b”.
